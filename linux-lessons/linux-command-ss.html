🌐 دستور ss – بررسی اتصالات شبکه (نسخه‌ی مدرن)

سطح: پیشرفته | زمان مطالعه: ۸ دقیقه

📖 نام (NAME)

ss – ابزار بررسی اتصالات شبکه (جایگزین netstat)

💻 خلاصه (SYNOPSIS)

ss [گزینه]

توضیح: ss (مخفف Socket Statistics) یک ابزار مدرن و سریع‌تر برای نمایش اطلاعات مربوط به سوکت‌ها و اتصالات شبکه است که جایگزین netstat شده است.

📖 توضیح کامل (DESCRIPTION)

ss اطلاعات دقیقی از سوکت‌ها، اتصالات، پورت‌های باز و جدول مسیریابی ارائه می‌دهد. این دستور بسیار سریع‌تر از netstat است و اطلاعات بیشتری را نمایش می‌دهد.

مزایای ss نسبت به netstat:

  • سرعت بالاتر (به‌ویژه در سیستم‌های با اتصالات زیاد).
  • اطلاعات دقیق‌تر.
  • پشتیبانی از فیلترهای پیشرفته.
  • پشتیبانی از TCP, UDP, UNIX sockets و ...

⚙️ لیست کامل گزینه‌ها (OPTIONS)

  • -t – نمایش اتصالات TCP.
  • -u – نمایش اتصالات UDP.
  • -l – نمایش پورت‌های در حال گوش‌دادن (Listening).
  • -a – نمایش همه‌ی اتصالات (Listening و غیر Listening).
  • -p – نمایش PID و نام برنامه.
  • -n – نمایش آدرس‌ها به صورت عددی (بدون تبدیل به نام).
  • -r – تبدیل آدرس‌های IP به نام (DNS).
  • -e – نمایش اطلاعات بیشتر (Extended).
  • -o – نمایش تایمرهای (timer) مربوط به اتصالات.
  • -s – نمایش خلاصه‌ی آماری.
  • -4 – فقط اتصالات IPv4.
  • -6 – فقط اتصالات IPv6.
  • --help – نمایش راهنمای کوتاه.
  • --version – نمایش نسخه‌ی دستور.

🧪 مثال‌ها (EXAMPLES)

مثال ۱: نمایش همه‌ی اتصالات TCP

ss -t
خروجی: 
State      Recv-Q Send-Q        Local Address:Port          Peer Address:Port
ESTAB      0      0             192.168.1.100:22            192.168.1.50:54321
LISTEN     0      128           0.0.0.0:22                  0.0.0.0:*
...

همه‌ی اتصالات TCP را نمایش می‌دهد.

مثال ۲: نمایش پورت‌های در حال گوش‌دادن (با PID)

sudo ss -tulpn
خروجی: 
Netid  State   Recv-Q  Send-Q  Local Address:Port   Peer Address:Port   Process
tcp    LISTEN  0       128     0.0.0.0:22           0.0.0.0:*           users:(("sshd",pid=1234,fd=3))
tcp    LISTEN  0       80      127.0.0.1:3306       0.0.0.0:*           users:(("mysqld",pid=5678,fd=14))
udp    UNCONN  0       0       0.0.0.0:68           0.0.0.0:*

پورت‌های در حال گوش‌دادن (TCP و UDP) را با PID و نام برنامه نمایش می‌دهد.

مثال ۳: نمایش اتصالات ESTABLISHED

ss -t state established
خروجی: 
(فقط اتصالات برقرار شده را نمایش می‌دهد)

با فیلتر state established، فقط اتصالات برقرار شده را نمایش می‌دهد.

مثال ۴: نمایش خلاصه‌ی آماری

ss -s
خروجی: 
Total: 120 (kernel 150)
TCP:   10 (estab 5, closed 2, orphaned 0, synrecv 0, timewait 0/0), ports 3

Transport Total     IP        IPv6
*         150       -         -        
RAW       0         0         0
UDP       2         2         0
TCP       10        10        0
...

خلاصه‌ی آماری اتصالات و سوکت‌ها را نمایش می‌دهد.

مثال ۵: نمایش همه‌ی اتصالات با جزئیات کامل

ss -tunap
خروجی: 
(همه‌ی اتصالات TCP و UDP با PID و نام برنامه، به صورت عددی)

ترکیب -tunap (TCP, UDP, عددی, همه, PID) یکی از پرکاربردترین ترکیب‌هاست.

🧠 نکته (BUGS & NOTES)

نکته: ss بسیار سریع‌تر از netstat است و اطلاعات بیشتری ارائه می‌دهد.

نکته: برای نمایش PID و نام برنامه، از -p و sudo استفاده کنید.

نکته: ss از فیلترهای پیشرفته‌ای مانند state و sport پشتیبانی می‌کند.

✍️ تمرین

تمرین ۱: با ss -tuln پورت‌های باز سیستم را ببینید.

تمرین ۲: با sudo ss -tulpn PID برنامه‌های در حال گوش‌دادن را ببینید.

تمرین ۳: با ss -s خلاصه‌ی آماری شبکه را ببینید.

تمرین ۴: با ss -t state established اتصالات برقرار شده را ببینید.
