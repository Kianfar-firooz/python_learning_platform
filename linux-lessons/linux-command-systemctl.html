⚙️ دستور systemctl – مدیریت سرویس‌ها

سطح: متوسط | زمان مطالعه: ۱۰ دقیقه

📖 نام (NAME)

systemctl – کنترل و مدیریت سرویس‌های سیستم (systemd)

💻 خلاصه (SYNOPSIS)

systemctl [گزینه] [دستور] [نام سرویس]

توضیح: systemctl ابزار مدیریت سرویس‌ها و سیستم‌عامل در توزیع‌های لینوکس مبتنی بر systemd (از جمله اوبونتو) است. این دستور برای شروع، توقف، فعال‌سازی و بررسی وضعیت سرویس‌ها استفاده می‌شود.

📖 توضیح کامل (DESCRIPTION)

systemctl ابزار اصلی مدیریت سرویس‌ها در لینوکس مدرن است. با این دستور می‌توانید سرویس‌های مختلف (مثل وب‌سرور، دیتابیس، SSH و ...) را مدیریت کنید.

دستورات اصلی:

  • start – شروع یک سرویس.
  • stop – توقف یک سرویس.
  • restart – راه‌اندازی مجدد سرویس.
  • reload – بارگذاری مجدد تنظیمات بدون توقف سرویس.
  • enable – فعال‌سازی سرویس برای شروع خودکار در زمان بوت.
  • disable – غیرفعال‌سازی شروع خودکار سرویس.
  • status – نمایش وضعیت سرویس.
  • is-active – بررسی فعال بودن سرویس.
  • is-enabled – بررسی فعال بودن شروع خودکار.
  • list-units – نمایش لیست سرویس‌های فعال.
  • list-unit-files – نمایش لیست تمام سرویس‌ها (فعال و غیرفعال).

⚙️ گزینه‌ها (OPTIONS)

  • --no-pager – نمایش خروجی بدون صفحه‌بندی.
  • --quiet – نمایش خروجی کمتر.
  • -f – اجرای اجباری (برای restart و reload).
  • -a – نمایش همه‌ی واحدها (شامل غیرفعال‌ها).
  • --help – نمایش راهنمای کوتاه.
  • --version – نمایش نسخه‌ی دستور.

🧪 مثال‌ها (EXAMPLES)

مثال ۱: مشاهده وضعیت یک سرویس

systemctl status ssh
خروجی: 
● ssh.service - OpenBSD Secure Shell server
   Loaded: loaded (/lib/systemd/system/ssh.service; enabled; vendor preset: enabled)
   Active: active (running) since Tue 2024-07-20 10:00:00 UTC
     Docs: man:sshd(8)
   Main PID: 1234 (sshd)
    Tasks: 1 (limit: 19143)
   Memory: 1.2M
   CGroup: /system.slice/ssh.service
           └─1234 /usr/sbin/sshd -D

وضعیت سرویس SSH را نمایش می‌دهد (فعال، غیرفعال، در حال اجرا و ...).

مثال ۲: شروع یک سرویس

sudo systemctl start apache2
خروجی: 
(هیچ خروجی، سرویس apache2 شروع می‌شود)

سرویس apache2 را شروع می‌کند.

مثال ۳: توقف یک سرویس

sudo systemctl stop apache2
خروجی: 
(هیچ خروجی، سرویس apache2 متوقف می‌شود)

سرویس apache2 را متوقف می‌کند.

مثال ۴: فعال‌سازی شروع خودکار یک سرویس

sudo systemctl enable apache2
خروجی: 
Created symlink /etc/systemd/system/multi-user.target.wants/apache2.service → /lib/systemd/system/apache2.service.

apache2 را برای شروع خودکار در زمان بوت فعال می‌کند.

مثال ۵: غیرفعال‌سازی شروع خودکار

sudo systemctl disable apache2
خروجی: 
Removed /etc/systemd/system/multi-user.target.wants/apache2.service.

شروع خودکار apache2 را غیرفعال می‌کند.

مثال ۶: راه‌اندازی مجدد یک سرویس

sudo systemctl restart apache2
خروجی: 
(هیچ خروجی، سرویس apache2 راه‌اندازی مجدد می‌شود)

apache2 را متوقف و دوباره شروع می‌کند.

🧠 نکته (BUGS & NOTES)

نکته: برای شروع/توقف سرویس‌ها معمولاً به دسترسی root نیاز دارید، پس از sudo استفاده کنید.

نکته: با systemctl list-units می‌توانید لیست سرویس‌های فعال را ببینید.

نکته: بعضی سرویس‌ها مانند nginx یا mysql ممکن است نام‌های متفاوتی داشته باشند.

✍️ تمرین

تمرین ۱: با systemctl status ssh وضعیت سرویس SSH را ببینید.

تمرین ۲: یک سرویس مثل cron را با systemctl start شروع کنید.

تمرین ۳: با systemctl enable شروع خودکار یک سرویس را فعال کنید.

تمرین ۴: با systemctl list-units لیست سرویس‌های فعال را ببینید.
