🌐 دستور ping – تست اتصال شبکه

سطح: متوسط | زمان مطالعه: ۸ دقیقه

📖 نام (NAME)

ping – ارسال بسته‌های ICMP برای تست اتصال به یک میزبان (Host) در شبکه

💻 خلاصه (SYNOPSIS)

ping [گزینه] [آدرس یا نام میزبان]

توضیح: ping یکی از اصلی‌ترین ابزارهای شبکه در لینوکس است که با ارسال بسته‌های ICMP Echo Request به یک میزبان، پاسخ آن را بررسی می‌کند. این دستور برای بررسی اتصال به اینترنت، سرورها و سایر دستگاه‌های شبکه بسیار مفید است.

📖 توضیح کامل (DESCRIPTION)

دستور ping به‌طور مداوم بسته‌های ICMP را به آدرس یا نام میزبان مورد نظر ارسال می‌کند و زمان رفت و برگشت (RTT) هر بسته را نمایش می‌دهد. این اطلاعات به شما کمک می‌کند تا:

  • مشخص کنید که آیا میزبان در دسترس است یا خیر.
  • سرعت پاسخ‌دهی (Latency) شبکه را اندازه‌گیری کنید.
  • مشکلات اتصال و افت بسته (Packet Loss) را تشخیص دهید.

⚙️ لیست کامل گزینه‌ها (OPTIONS)

  • -c [count] – تعداد بسته‌های ارسالی را مشخص می‌کند (پس از آن متوقف می‌شود).
  • -i [seconds] – فاصله‌ی زمانی بین ارسال بسته‌ها (پیش‌فرض ۱ ثانیه).
  • -s [size] – اندازه‌ی بسته را مشخص می‌کند (به بایت).
  • -t [ttl] – مقدار TTL (Time To Live) را تنظیم می‌کند.
  • -W [timeout] – زمان انتظار برای پاسخ (به ثانیه).
  • -4 – استفاده از IPv4.
  • -6 – استفاده از IPv6.
  • -q – نمایش خلاصه (quiet mode).
  • -v – نمایش جزئیات بیشتر (verbose).
  • --help – نمایش راهنمای کوتاه.
  • --version – نمایش نسخه‌ی دستور.

🧪 مثال‌ها (EXAMPLES)

مثال ۱: تست اتصال به گوگل (نام دامنه)

ping google.com
خروجی: 
PING google.com (142.250.185.78) 56(84) bytes of data.
64 bytes from 142.250.185.78: icmp_seq=1 ttl=117 time=15.2 ms
64 bytes from 142.250.185.78: icmp_seq=2 ttl=117 time=14.8 ms
64 bytes from 142.250.185.78: icmp_seq=3 ttl=117 time=15.0 ms
^C
--- google.com ping statistics ---
3 packets transmitted, 3 received, 0% packet loss, time 2003ms
rtt min/avg/max/mdev = 14.800/15.000/15.200/0.200 ms

ping را با Ctrl+C متوقف می‌کنید. خروجی آمار نهایی را نمایش می‌دهد.

مثال ۲: ارسال ۴ بسته و توقف خودکار

ping -c 4 google.com
خروجی: 
PING google.com (142.250.185.78) 56(84) bytes of data.
64 bytes from 142.250.185.78: icmp_seq=1 ttl=117 time=15.2 ms
64 bytes from 142.250.185.78: icmp_seq=2 ttl=117 time=14.8 ms
64 bytes from 142.250.185.78: icmp_seq=3 ttl=117 time=15.0 ms
64 bytes from 142.250.185.78: icmp_seq=4 ttl=117 time=15.1 ms

--- google.com ping statistics ---
4 packets transmitted, 4 received, 0% packet loss, time 3005ms
rtt min/avg/max/mdev = 14.800/15.025/15.200/0.150 ms

با -c 4 فقط ۴ بسته ارسال می‌شود و برنامه خودکار متوقف می‌شود.

مثال ۳: تست اتصال به یک آدرس IP

ping 8.8.8.8
خروجی: 
PING 8.8.8.8 (8.8.8.8) 56(84) bytes of data.
64 bytes from 8.8.8.8: icmp_seq=1 ttl=117 time=15.2 ms
64 bytes from 8.8.8.8: icmp_seq=2 ttl=117 time=14.8 ms
...

آدرس IP سرور DNS گوگل (8.8.8.8) را پینگ می‌کند.

مثال ۴: تغییر فاصله ارسال بسته‌ها

ping -i 0.5 google.com
خروجی: 
(بسته‌ها هر ۰.۵ ثانیه ارسال می‌شوند)

فاصله‌ی ارسال بسته‌ها را به ۰.۵ ثانیه کاهش می‌دهد.

🧠 نکته (BUGS & NOTES)

نکته: اگر ping پاسخی دریافت نکند، ممکن است میزبان خاموش باشد یا فایروال درخواست‌های ICMP را مسدود کرده باشد.

نکته: برای پینگ کردن به‌طور مداوم، از گزینه‌های -c استفاده نکنید و با Ctrl+C آن را متوقف کنید.

نکته: زمان پاسخ (Time) نشان‌دهنده‌ی سرعت شبکه است. هرچه کمتر باشد، اتصال سریع‌تر است.

✍️ تمرین

تمرین ۱: با ping اتصال خود به سرور google.com را تست کنید.

تمرین ۲: با ping -c 5 فقط ۵ بسته ارسال کنید.

تمرین ۳: آدرس IP یک سرور دیگر (مثل 1.1.1.1) را پینگ کنید.

تمرین ۴: با ping -i 2 فاصله‌ی ارسال را به ۲ ثانیه افزایش دهید.
