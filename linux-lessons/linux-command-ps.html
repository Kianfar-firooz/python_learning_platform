🔄 دستور ps – نمایش فرآیندهای در حال اجرا

سطح: متوسط | زمان مطالعه: ۱۰ دقیقه

📖 نام (NAME)

ps – گزارش از فرآیندهای (processes) در حال اجرا در سیستم

💻 خلاصه (SYNOPSIS)

ps [گزینه]

توضیح: اطلاعات مربوط به فرآیندهای در حال اجرا در سیستم را نمایش می‌دهد. این دستور برای مدیریت و عیب‌یابی فرآیندها بسیار مفید است.

📖 توضیح کامل (DESCRIPTION)

دستور ps (مخفف Process Status) لیستی از فرآیندهای در حال اجرا در سیستم را نمایش می‌دهد. این فرآیندها شامل برنامه‌های در حال اجرا، سرویس‌های سیستمی و دستوراتی هستند که در پس‌زمینه اجرا می‌شوند.

ps اطلاعات مختلفی مانند شناسه فرآیند (PID)، نام فرآیند، استفاده از CPU و حافظه، وضعیت فرآیند و کاربر اجراکننده را نمایش می‌دهد.

⚙️ لیست کامل گزینه‌ها (OPTIONS)

  • -e یا -A – نمایش همه‌ی فرآیندها (همه‌ی کاربران).
  • -f – نمایش اطلاعات کامل (full format).
  • -u [user] – نمایش فرآیندهای یک کاربر خاص.
  • -p [PID] – نمایش اطلاعات یک فرآیند خاص با شناسه‌ی PID.
  • --forest – نمایش درختی فرآیندها (ارتباط والد-فرزند).
  • -o [format] – انتخاب ستون‌های خروجی (مثلاً -o pid,comm,%cpu).
  • aux – نمایش همه‌ی فرآیندها با جزئیات کامل (فرمت BSD).
  • ef – نمایش همه‌ی فرآیندها با جزئیات کامل (فرمت System V).
  • --help – نمایش راهنمای کوتاه.
  • --version – نمایش نسخه‌ی دستور.

📖 ستون‌های مهم در خروجی

  • PID: شناسه‌ی فرآیند
  • PPID: شناسه‌ی فرآیند والد (که این فرآیند را اجرا کرده است)
  • USER: کاربری که فرآیند را اجرا کرده است
  • %CPU: درصد استفاده از CPU
  • %MEM: درصد استفاده از حافظه
  • VSZ: حجم حافظه مجازی (به کیلوبایت)
  • RSS: حجم حافظه فیزیکی (به کیلوبایت)
  • STAT: وضعیت فرآیند (مثل R = در حال اجرا، S = در حال استراحت، Z = زامبی)
  • START: زمان شروع فرآیند
  • TIME: زمان استفاده از CPU
  • COMMAND: نام دستور یا برنامه

🧪 مثال‌ها (EXAMPLES)

مثال ۱: نمایش فرآیندهای کاربر فعلی

ps
خروجی: 
  PID TTY          TIME CMD
 1234 pts/0    00:00:00 bash
 4567 pts/0    00:00:00 ps

فرآیندهای متعلق به کاربر فعلی را نمایش می‌دهد.

مثال ۲: نمایش همه‌ی فرآیندها با جزئیات کامل

ps aux
خروجی: 
USER       PID %CPU %MEM    VSZ   RSS TTY      STAT START   TIME COMMAND
root         1  0.0  0.1  22500  1200 ?        Ss   10:00   0:00 /sbin/init
user      1234  0.0  0.2  24500  2500 pts/0    Ss   10:01   0:00 bash
user      4567  0.0  0.1  27500  1500 pts/0    R+   10:02   0:00 ps aux

همه‌ی فرآیندهای سیستم را با جزئیات کامل نمایش می‌دهد.

مثال ۳: نمایش فرآیندهای یک کاربر خاص

ps -u alice
خروجی: 
  PID TTY          TIME CMD
 5678 pts/1    00:00:00 bash
 7890 pts/1    00:00:01 python3

فقط فرآیندهای متعلق به کاربر alice را نمایش می‌دهد.

مثال ۴: نمایش درختی فرآیندها

ps --forest
خروجی: 
  PID TTY          TIME CMD
 1234 pts/0    00:00:00 bash
 4567 pts/0    00:00:00  \_ ps --forest

فرآیندها را به صورت درختی نمایش می‌دهد تا ارتباط والد-فرزند مشخص شود.

مثال ۵: انتخاب ستون‌های خاص

ps -o pid,comm,%cpu,%mem --sort=-%cpu
خروجی: 
  PID COMMAND         %CPU %MEM
 1234 bash             0.0  0.2
 4567 ps               0.0  0.1

فقط ستون‌های pid، comm، %cpu و %mem را نمایش می‌دهد و بر اساس مصرف CPU مرتب‌سازی می‌کند.

🧠 نکته (BUGS & NOTES)

نکته: ترکیب aux یکی از پرکاربردترین گزینه‌های ps است و همه‌ی فرآیندها را با جزئیات کامل نمایش می‌دهد.

نکته: برای مانیتورینگ پویا (زنده) از دستور top یا htop استفاده کنید.

نکته: برای پیدا کردن PID یک فرآیند خاص، از ترکیب ps aux | grep [name] استفاده کنید.

✍️ تمرین

تمرین ۱: با ps فرآیندهای کاربر فعلی را ببینید.

تمرین ۲: با ps aux همه‌ی فرآیندهای سیستم را با جزئیات کامل مشاهده کنید.

تمرین ۳: با ps -u فرآیندهای یک کاربر خاص را ببینید.

تمرین ۴: با ps --forest ساختار درختی فرآیندها را مشاهده کنید.

تمرین ۵: با ps -o ستون‌های دلخواه خود را انتخاب کنید.
