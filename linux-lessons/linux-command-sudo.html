🔧 دستور sudo – اجرای دستورات با سطح دسترسی مدیر

سطح: مقدماتی | زمان مطالعه: ۸ دقیقه

📖 نام (NAME)

sudo – اجرای یک دستور به عنوان کاربر دیگر (معمولاً مدیر سیستم)

💻 خلاصه (SYNOPSIS)

sudo [گزینه] [دستور]

توضیح: sudo به شما اجازه می‌دهد یک دستور را با سطح دسترسی مدیر سیستم (root) یا کاربر دیگر اجرا کنید. این دستور برای کارهای مدیریتی مانند نصب برنامه، تغییر تنظیمات سیستمی و مدیریت فایل‌های محافظت‌شده استفاده می‌شود.

📖 توضیح کامل (DESCRIPTION)

sudo مخفف SuperUser DO است و به کاربران مجاز اجازه می‌دهد تا دستورات را با سطح دسترسی root (مدیر سیستم) اجرا کنند. کاربر برای استفاده از sudo باید در گروه sudo باشد. معمولاً اولین بار که از sudo استفاده می‌کنید، از شما رمز عبور خودتان را می‌خواهد (نه رمز root).

این دستور برای انجام کارهایی که نیاز به دسترسی بالا دارند، مثل نصب یا حذف برنامه‌ها، تغییر فایل‌های سیستمی و مدیریت سرویس‌ها، ضروری است.

⚙️ لیست کامل گزینه‌ها (OPTIONS)

  • -u [user] – اجرای دستور با کاربر دیگر (به جز root).
  • -g [group] – اجرای دستور با گروه مشخص.
  • -s – اجرای یک شل (shell) با سطح دسترسی root.
  • -i – اجرای شل با محیط کاربر root (login shell).
  • -k – غیرفعال کردن اعتبار sudo (برای دفعات بعد نیاز به رمز دارد).
  • -K – حذف کامل اعتبار sudo (مشابه -k اما مطمئن‌تر).
  • -v – تمدید اعتبار sudo (بدون اجرای دستور).
  • -l – نمایش لیست دستوراتی که کاربر فعلی می‌تواند با sudo اجرا کند.
  • -b – اجرای دستور در پس‌زمینه (background).
  • -H – تنظیم متغیر HOME به پوشه‌ی خانه‌ی کاربر هدف.
  • -E – حفظ متغیرهای محیطی فعلی.
  • --help – نمایش راهنمای کوتاه.
  • --version – نمایش نسخه‌ی دستور.

🧪 مثال‌ها (EXAMPLES)

مثال ۱: اجرای یک دستور با sudo

ساده‌ترین حالت استفاده از sudo برای اجرای دستوری که نیاز به دسترسی root دارد:

sudo apt update
خروجی: 
[sudo] password for user: 
(رمز عبور خود را وارد کنید)
Hit:1 http://archive.ubuntu.com/ubuntu focal InRelease
...

دستور apt update را با سطح دسترسی root اجرا می‌کند.

مثال ۲: اجرای دستور با کاربر دیگر

با گزینه‌ی -u می‌توانید دستور را با کاربر دیگری اجرا کنید:

sudo -u www-data ls /var/www
خروجی: 
html
index.html

دستور ls /var/www را با کاربر www-data اجرا می‌کند.

مثال ۳: اجرای شل به عنوان root

با گزینه‌ی -s یک شل (shell) با سطح دسترسی root باز می‌شود:

sudo -s
خروجی: 
[sudo] password for user:
root@ubuntu:~#

شل را با کاربر root باز می‌کند. برای خروج، exit یا Ctrl+D بزنید.

⚠️ هشدار: با دقت از این حالت استفاده کنید، زیرا همه‌ی دستورات با سطح دسترسی کامل اجرا می‌شوند.

مثال ۴: نمایش دستورات قابل اجرا با sudo

با گزینه‌ی -l می‌توانید ببینید چه دستوراتی را با sudo می‌توانید اجرا کنید:

sudo -l
خروجی: 
User user may run the following commands on ubuntu:
    (ALL : ALL) ALL

این خروجی نشان می‌دهد که کاربر می‌تواند همه‌ی دستورات را با sudo اجرا کند.

مثال ۵: تمدید اعتبار sudo

با گزینه‌ی -v می‌توانید اعتبار sudo را بدون اجرای دستور تمدید کنید:

sudo -v
خروجی: 
[sudo] password for user:
(رمز را وارد کنید، سپس اعتبار تمدید می‌شود)

زمان اعتبار sudo را تا ۱۵ دقیقه دیگر تمدید می‌کند.

🧠 نکته (BUGS & NOTES)

نکته مهم: از sudo با احتیاط استفاده کنید، زیرا دستورات با سطح دسترسی root می‌توانند به سیستم آسیب بزنند. همیشه قبل از اجرای دستورات خطرناک، مطمئن شوید که درست تایپ کرده‌اید.

نکته: برای اجرای دستورات طولانی یا زنجیره‌ای، از sudo فقط یک بار در ابتدای زنجیره استفاده کنید.

نکته: اگر نیاز به اجرای چندین دستور با sudo دارید، از sudo -s یا sudo -i استفاده کنید تا یک شل root باز شود.

✍️ تمرین

تمرین ۱: با sudo دستور apt update را اجرا کنید.

تمرین ۲: با sudo -l ببینید چه دستوراتی را می‌توانید با sudo اجرا کنید.

تمرین ۳: با sudo -u یک دستور را با کاربر دیگری اجرا کنید (مثلاً whoami).
