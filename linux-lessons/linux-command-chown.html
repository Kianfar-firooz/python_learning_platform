👤 دستور chown – تغییر مالکیت فایل‌ها و پوشه‌ها

سطح: متوسط | زمان مطالعه: ۱۰ دقیقه

📖 نام (NAME)

chown – تغییر مالک (owner) و گروه (group) فایل‌ها و پوشه‌ها

💻 خلاصه (SYNOPSIS)

chown [گزینه] [مالک][:گروه] [فایل/پوشه]

توضیح: مالک و/یا گروه یک فایل یا پوشه را تغییر می‌دهد. برای تغییر مالکیت فایل‌هایی که مالک آن‌ها نیستید، باید از sudo استفاده کنید.

📖 توضیح کامل (DESCRIPTION)

دستور chown (مخفف Change Owner) برای تغییر مالک (user) و/یا گروه (group) یک فایل یا پوشه استفاده می‌شود. هر فایل در لینوکس دارای یک مالک و یک گروه است که تعیین می‌کند چه کسی به آن دسترسی دارد.

این دستور معمولاً توسط مدیر سیستم (با استفاده از sudo) برای تغییر مالکیت فایل‌ها و پوشه‌ها استفاده می‌شود.

⚙️ لیست کامل گزینه‌ها (OPTIONS)

  • -R – تغییر مالکیت به صورت بازگشتی (برای پوشه‌ها و زیرپوشه‌ها).
  • -v – نمایش جزئیات تغییرات (verbose).
  • -c – فقط تغییرات اعمال‌شده را نمایش می‌دهد.
  • -f – نمایش خطاها را غیرفعال می‌کند.
  • --help – نمایش راهنمای کوتاه.
  • --version – نمایش نسخه‌ی دستور.

📖 روش‌های استفاده

  • chown user file – فقط مالک را تغییر می‌دهد.
  • chown user:group file – مالک و گروه را تغییر می‌دهد.
  • chown :group file – فقط گروه را تغییر می‌دهد.
  • chown user: file – مالک را تغییر می‌دهد و گروه را به گروه پیش‌فرض مالک تنظیم می‌کند.

🧪 مثال‌ها (EXAMPLES)

مثال ۱: تغییر مالک یک فایل

sudo chown alice file.txt
خروجی: 
(هیچ خروجی، مالک فایل به alice تغییر می‌کند)

مالک file.txt را به کاربر alice تغییر می‌دهد.

مثال ۲: تغییر مالک و گروه

sudo chown alice:developers file.txt
خروجی: 
(هیچ خروجی، مالک و گروه تغییر می‌کنند)

مالک را به alice و گروه را به developers تغییر می‌دهد.

مثال ۳: تغییر فقط گروه

sudo chown :developers file.txt
خروجی: 
(هیچ خروجی، فقط گروه تغییر می‌کند)

فقط گروه فایل را به developers تغییر می‌دهد (مالک تغییر نمی‌کند).

مثال ۴: تغییر مالکیت یک پوشه و محتویات آن

sudo chown -R alice:developers myfolder
خروجی: 
(مالکیت all محتویات پوشه تغییر می‌کند)

مالک و گروه پوشه‌ی myfolder و تمام فایل‌ها و زیرپوشه‌های آن را به alice:developers تغییر می‌دهد.

مثال ۵: تغییر مالکیت با نمایش جزئیات

sudo chown -v alice file.txt
خروجی: 
changed ownership of 'file.txt' from root to alice

با گزینه‌ی -v، تغییرات اعمال‌شده را نمایش می‌دهد.

🧠 نکته (BUGS & NOTES)

نکته: برای تغییر مالکیت فایل‌هایی که مالک آن‌ها نیستید، باید از sudo استفاده کنید.

نکته: با chown نمی‌توانید مالکیت یک فایل را به کاربری که وجود ندارد تغییر دهید. کاربر و گروه باید از قبل در سیستم تعریف شده باشند.

نکته: برای دیدن مالک و گروه فعلی یک فایل، از ls -l استفاده کنید.

✍️ تمرین

تمرین ۱: یک فایل بسازید و با ls -l مالک و گروه آن را ببینید.

تمرین ۲: با sudo chown مالک فایل را به کاربر دیگری تغییر دهید (اگر کاربر دیگری ندارید، از root استفاده کنید).

تمرین ۳: با sudo chown : گروه فایل را تغییر دهید.

تمرین ۴: با sudo chown -R مالکیت یک پوشه و محتویات آن را تغییر دهید.
