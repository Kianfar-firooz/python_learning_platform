🔍 دستور grep – جستجوی الگو در متن

سطح: متوسط | زمان مطالعه: ۱۲ دقیقه

📖 نام (NAME)

grep – جستجوی الگوهای متنی در فایل‌ها یا ورودی استاندارد

💻 خلاصه (SYNOPSIS)

grep [گزینه] [الگو] [فایل]

توضیح: یک الگو (مثلاً یک کلمه یا عبارت) را در فایل‌ها یا خروجی دستورات جستجو می‌کند و خطوطی که شامل آن الگو هستند را نمایش می‌دهد. این یکی از قدرتمندترین دستورات لینوکس برای پردازش متن است.

📖 توضیح کامل (DESCRIPTION)

دستور grep (مخفف Global Regular Expression Print) برای جستجوی الگوهای متنی در فایل‌ها یا ورودی استاندارد استفاده می‌شود. این دستور با استفاده از عبارت‌های منظم (Regular Expressions) می‌تواند جستجوهای پیچیده‌ای انجام دهد.

grep به‌طور پیش‌فرض به حروف بزرگ و کوچک حساس است، اما با گزینه‌های مناسب می‌توان این حساسیت را غیرفعال کرد.

⚙️ لیست کامل گزینه‌ها (OPTIONS)

  • -i – جستجوی غیرحساس به حروف بزرگ و کوچک (ignore case).
  • -v – نمایش خطوطی که الگو را ندارند (invert match).
  • -r یا -R – جستجوی بازگشتی در پوشه‌ها و زیرپوشه‌ها.
  • -n – نمایش شماره خطوط همراه با نتایج.
  • -c – فقط تعداد خطوط منطبق را نمایش می‌دهد (count).
  • -l – فقط نام فایل‌های منطبق را نمایش می‌دهد (files with match).
  • -w – جستجوی کلمه‌ی کامل (whole word).
  • -x – جستجوی خط کامل (whole line).
  • -A [num] – نمایش num خط بعد از خط منطبق.
  • -B [num] – نمایش num خط قبل از خط منطبق.
  • -C [num] – نمایش num خط قبل و بعد از خط منطبق.
  • -e [pattern] – استفاده از چندین الگو.
  • -f [file] – خواندن الگوها از یک فایل.
  • -E – استفاده از عبارت‌های منظم توسعه‌یافته (Extended Regex).
  • -F – جستجوی رشته‌ی ثابت (بدون تفسیر regex).
  • --color – نمایش نتایج با رنگ.
  • --help – نمایش راهنمای کوتاه.
  • --version – نمایش نسخه‌ی دستور.

🧪 مثال‌ها (EXAMPLES)

مثال ۱: جستجوی یک کلمه در یک فایل

grep "error" log.txt
خروجی: 
[2024-07-20 10:00:00] error: connection failed
[2024-07-20 10:05:00] error: timeout

خطوطی که شامل کلمه‌ی "error" هستند را از فایل log.txt نمایش می‌دهد.

مثال ۲: جستجوی غیرحساس به حروف بزرگ و کوچک

grep -i "error" log.txt
خروجی: 
[2024-07-20 10:00:00] ERROR: connection failed
[2024-07-20 10:05:00] error: timeout

هم "error" و هم "ERROR" و هر ترکیب دیگری را پیدا می‌کند.

مثال ۳: نمایش خطوطی که الگو را ندارند

grep -v "error" log.txt
خروجی: 
[2024-07-20 09:55:00] info: system started
[2024-07-20 10:10:00] info: task completed

خطوطی را نمایش می‌دهد که شامل کلمه‌ی "error" نیستند.

مثال ۴: جستجوی بازگشتی در یک پوشه

grep -r "TODO" src/
خروجی: 
src/main.py:10: # TODO: implement this function
src/utils.py:25: # TODO: add error handling

کلمه‌ی "TODO" را در تمام فایل‌های پوشه‌ی src/ و زیرپوشه‌های آن جستجو می‌کند.

مثال ۵: نمایش شماره خطوط

grep -n "error" log.txt
خروجی: 
5:[2024-07-20 10:00:00] error: connection failed
12:[2024-07-20 10:05:00] error: timeout

همراه با شماره خطوط، نتایج را نمایش می‌دهد.

مثال ۶: استفاده از grep با خروجی دستور دیگر

ps aux | grep "python"
خروجی: 
user   1234  0.0  0.5  12345  6789 pts/0    S+   10:00   0:00 python3 script.py

خروجی ps aux را فیلتر می‌کند و فقط فرآیندهای حاوی "python" را نمایش می‌دهد.

🧠 نکته (BUGS & NOTES)

نکته: grep به‌طور پیش‌فرض به حروف بزرگ و کوچک حساس است. برای جستجوی غیرحساس، از -i استفاده کنید.

نکته: برای جستجوی عبارت‌های با فاصله، آن‌ها را درون نقل قول (") قرار دهید.

نکته: برای جستجوی پیشرفته‌تر، از عبارت‌های منظم (Regular Expressions) با -E استفاده کنید.

✍️ تمرین

تمرین ۱: یک فایل متنی بسازید و چند خط در آن بنویسید. با grep یک کلمه را در آن جستجو کنید.

تمرین ۲: با grep -i جستجوی غیرحساس را امتحان کنید.

تمرین ۳: با grep -v خطوطی که کلمه‌ی مورد نظر را ندارند، نمایش دهید.

تمرین ۴: با ps aux | grep فرآیندهای مربوط به یک برنامه را پیدا کنید.
