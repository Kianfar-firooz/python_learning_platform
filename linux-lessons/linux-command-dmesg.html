📋 دستور dmesg – مشاهده لاگ‌های هسته (Kernel logs)

سطح: پیشرفته | زمان مطالعه: ۸ دقیقه

📖 نام (NAME)

dmesg – نمایش پیام‌های بافر حلقوی (Ring Buffer) هسته لینوکس

💻 خلاصه (SYNOPSIS)

dmesg [گزینه]

توضیح: dmesg (مخفف Diagnostic Message) لاگ‌های مربوط به هسته (Kernel) لینوکس را نمایش می‌دهد. این لاگ‌ها شامل اطلاعات بوت، شناسایی سخت‌افزار، خطاهای سیستمی و ... هستند.

📖 توضیح کامل (DESCRIPTION)

dmesg یکی از ابزارهای اصلی برای عیب‌یابی و بررسی سیستم در زمان بوت و پس از آن است. این دستور اطلاعات زیر را نمایش می‌دهد:

  • پیام‌های بوت (Boot messages).
  • تشخیص و شناسایی سخت‌افزارها.
  • خطاهای مربوط به درایورها.
  • اطلاعات مربوط به حافظه و CPU.
  • وضعیت دستگاه‌های متصل.

این لاگ‌ها برای مدیران سیستم و عیب‌یاب‌ها بسیار ارزشمند هستند.

⚙️ لیست کامل گزینه‌ها (OPTIONS)

  • -H – نمایش خروجی به صورت قابل خواندن (Human-readable).
  • -T – نمایش زمان‌ها به صورت قابل خواندن (با تاریخ و ساعت).
  • -c – نمایش و پاک کردن بافر (clear after display).
  • -f [facility] – فیلتر کردن بر اساس نوع پیام (مثل -f kern).
  • -l [level] – فیلتر کردن بر اساس سطح (مثل -l err برای خطاها).
  • -n [level] – تنظیم سطح گزارش‌دهی (log level).
  • -s [size] – تنظیم اندازه‌ی بافر برای خواندن.
  • -w – نمایش لاگ‌ها به‌صورت زنده (مانند tail -f).
  • --help – نمایش راهنمای کوتاه.
  • --version – نمایش نسخه‌ی دستور.

🧪 مثال‌ها (EXAMPLES)

مثال ۱: نمایش همه‌ی لاگ‌های هسته

dmesg
خروجی (نمونه): 
[    0.000000] Linux version 5.15.0-91-generic (buildd@lcy02-amd64-001) (gcc (Ubuntu 11.4.0-1ubuntu1~22.04) 11.4.0, GNU ld (GNU Binutils for Ubuntu) 2.38) #101-Ubuntu SMP Tue Nov 15 18:00:00 UTC 2024 (Ubuntu 5.15.0-91.101-generic 5.15.148)
[    0.000000] Command line: BOOT_IMAGE=/vmlinuz-5.15.0-91-generic root=UUID=abc123 ro quiet splash
[    0.000000] KERNEL supported cpus:
[    0.000000]   Intel GenuineIntel
[    0.000000]   AMD AuthenticAMD
[    0.000000]   Hygon HygonGenuine
[    0.000000]   Centaur CentaurHauls
[    0.000000]   zhaoxin   Shanghai  
...

همه‌ی پیام‌های هسته از زمان بوت تا کنون را نمایش می‌دهد.

مثال ۲: نمایش لاگ‌ها با زمان قابل خواندن

dmesg -T
خروجی: 
[Sun Jul 21 10:00:00 2024] Linux version 5.15.0-91-generic ...
[Sun Jul 21 10:00:01 2024] Command line: BOOT_IMAGE=...
...

لاگ‌ها را با تاریخ و ساعت قابل خواندن نمایش می‌دهد.

مثال ۳: فیلتر کردن خطاها

dmesg -l err
خروجی: 
(فقط پیام‌های خطا نمایش داده می‌شوند)

فقط پیام‌های با سطح err (خطا) را نمایش می‌دهد.

مثال ۴: نمایش لاگ‌های مربوط به دستگاه‌های USB

dmesg | grep USB
خروجی: 
[    1.234567] usb 1-1: new high-speed USB device number 2 using ehci-pci
[    1.345678] usb 1-1: New USB device found, idVendor=abcd, idProduct=1234
...

با ترکیب dmesg | grep می‌توانید لاگ‌های مربوط به یک دستگاه خاص را پیدا کنید.

مثال ۵: نمایش زنده‌ی لاگ‌ها

dmesg -w
خروجی: 
(لاگ‌های جدید به‌صورت لحظه‌ای نمایش داده می‌شوند)

مانند tail -f، لاگ‌های هسته را به‌صورت زنده نمایش می‌دهد.

🧠 نکته (BUGS & NOTES)

نکته: dmesg برای عیب‌یابی مشکلات سخت‌افزاری و خطاهای بوت بسیار مفید است.

نکته: بافر هسته محدود است، بنابراین لاگ‌های قدیمی ممکن است از دست بروند.

نکته: برای مشاهده‌ی لاگ‌های کامل سیستم، از journalctl استفاده کنید (در سیستم‌های با systemd).

✍️ تمرین

تمرین ۱: با dmesg لاگ‌های هسته را ببینید.

تمرین ۲: با dmesg -T لاگ‌ها را با زمان قابل خواندن ببینید.

تمرین ۳: با dmesg | grep پیام‌های مربوط به error را پیدا کنید.

تمرین ۴: با dmesg -w لاگ‌ها را به‌صورت زنده دنبال کنید.
