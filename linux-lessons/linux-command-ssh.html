🔐 دستور ssh – اتصال امن به سرور از راه دور

سطح: پیشرفته | زمان مطالعه: ۱۲ دقیقه

📖 نام (NAME)

ssh – پروتکل SSH (Secure Shell) برای اتصال امن به سیستم‌های از راه دور

💻 خلاصه (SYNOPSIS)

ssh [گزینه] [کاربر@]میزبان

توضیح: ssh (مخفف Secure Shell) یک پروتکل شبکه امن برای اتصال به سیستم‌های از راه دور است. این دستور به شما اجازه می‌دهد تا به یک سرور لینوکس متصل شده و روی آن کار کنید.

📖 توضیح کامل (DESCRIPTION)

ssh یکی از مهم‌ترین ابزارهای مدیریت سرور است که به شما امکان می‌دهد:

  • به سرورهای لینوکس متصل شوید.
  • دستورات را روی سرور اجرا کنید.
  • فایل‌ها را به‌صورت امن انتقال دهید (با scp و rsync).
  • پورت‌های محلی را به سرور تونل کنید (Port Forwarding).

اتصال SSH به‌طور پیش‌فرض از پورت ۲۲ استفاده می‌کند و با رمزنگاری قوی، امنیت ارتباط را تضمین می‌کند.

⚙️ لیست کامل گزینه‌ها (OPTIONS)

  • -p [port] – تنظیم پورت (اگر سرور روی پورت غیر از ۲۲ باشد).
  • -i [keyfile] – استفاده از کلید خصوصی (Private Key).
  • -v – نمایش جزئیات (Verbose – برای عیب‌یابی).
  • -q – حالت بی‌صدا (Quiet).
  • -C – فعال‌سازی فشرده‌سازی (Compression).
  • -X – فوروارد کردن X11 (برای اجرای برنامه‌های گرافیکی).
  • -L [local:remote] – فوروارد کردن پورت محلی به سرور.
  • -R [remote:local] – فوروارد کردن پورت سرور به محلی.
  • -o [option] – تنظیم گزینه‌های پیشرفته (مثل -o StrictHostKeyChecking=no).
  • --help – نمایش راهنمای کوتاه.
  • --version – نمایش نسخه‌ی دستور.

🧪 مثال‌ها (EXAMPLES)

مثال ۱: اتصال به یک سرور

ssh user@192.168.1.100
خروجی: 
The authenticity of host '192.168.1.100 (192.168.1.100)' can't be established.
ECDSA key fingerprint is SHA256:...
Are you sure you want to continue connecting (yes/no/[fingerprint])? yes
Warning: Permanently added '192.168.1.100' (ECDSA) to the list of known hosts.
user@192.168.1.100's password: 
(پس از وارد کردن رمز عبور، به سرور متصل می‌شوید)
user@server:~$

با کاربر user به سرور با IP 192.168.1.100 متصل می‌شوید. اولین بار از شما تأیید می‌خواهد.

مثال ۲: اتصال با پورت غیر استاندارد

ssh -p 2222 user@example.com
خروجی: 
(اتصال با پورت ۲۲۲۲ به جای پورت پیش‌فرض ۲۲)

اگر سرور روی پورت ۲۲۲۲ کار می‌کند، از -p استفاده کنید.

مثال ۳: اجرای یک دستور روی سرور (بدون ورود به شل)

ssh user@example.com ls -l /var/log
خروجی: 
total 1024
-rw-r--r-- 1 root root  12345 Jul 20 10:00 syslog
-rw-r--r-- 1 root root   6789 Jul 19 08:00 auth.log

دستور ls -l /var/log را روی سرور اجرا کرده و خروجی آن را نمایش می‌دهد.

مثال ۴: استفاده از کلید خصوصی

ssh -i ~/.ssh/id_rsa user@example.com
خروجی: 
(اتصال با استفاده از کلید SSH، بدون نیاز به رمز عبور)

با کلید خصوصی ~/.ssh/id_rsa به سرور متصل می‌شود.

مثال ۵: کپی کردن کلید عمومی به سرور

برای اتصال بدون رمز عبور، کلید عمومی خود را به سرور کپی کنید:

ssh-copy-id user@example.com
خروجی: 
/usr/bin/ssh-copy-id: INFO: Source of key(s) to be installed: "/home/user/.ssh/id_rsa.pub"
/usr/bin/ssh-copy-id: INFO: attempting to log in with the new key(s), to filter out any that are already installed
/usr/bin/ssh-copy-id: INFO: 1 key(s) remain to be installed -- if you are prompted now it is to install the new keys
user@example.com's password: 
(پس از وارد کردن رمز عبور، کلید کپی می‌شود)

کلید عمومی شما را به سرور کپی می‌کند تا دفعات بعد بدون رمز عبور وارد شوید.

🧠 نکته (BUGS & NOTES)

نکته: برای امنیت بیشتر، اتصال SSH را با کلید (Key-based authentication) انجام دهید و از ورود با رمز عبور خودداری کنید.

نکته: از ssh -v برای عیب‌یابی اتصال استفاده کنید.

نکته: فایل ~/.ssh/known_hosts شامل کلیدهای سرورهایی است که قبلاً به آن‌ها متصل شده‌اید.

✍️ تمرین

تمرین ۱: اگر به یک سرور دسترسی دارید، با ssh به آن متصل شوید.

تمرین ۲: یک جفت کلید SSH با ssh-keygen بسازید.

تمرین ۳: با ssh-copy-id کلید عمومی خود را به یک سرور کپی کنید.

تمرین ۴: با ssh -p به یک سرور با پورت غیر استاندارد متصل شوید.
