🌐 دستور curl – انتقال داده از/به سرور

سطح: متوسط | زمان مطالعه: ۱۰ دقیقه

📖 نام (NAME)

curl – ابزار انتقال داده برای URLها (پشتیبانی از HTTP, HTTPS, FTP و ...)

💻 خلاصه (SYNOPSIS)

curl [گزینه] [URL]

توضیح: curl یکی از قدرتمندترین ابزارهای خط فرمان برای انتقال داده از/به سرورها است. این دستور از پروتکل‌های مختلفی مانند HTTP، HTTPS، FTP، SFTP و ... پشتیبانی می‌کند و برای تست API، دانلود فایل و ... بسیار مفید است.

📖 توضیح کامل (DESCRIPTION)

دستور curl یک ابزار بسیار قدرتمند و انعطاف‌پذیر است که به شما اجازه می‌دهد با سرورها ارتباط برقرار کنید. این دستور می‌تواند محتوای یک وب‌سایت را دریافت کند، فایل‌ها را دانلود یا آپلود کند، و با APIهای وب کار کند.

curl مخفف "Client URL" است و در اسکریپت‌ها و برنامه‌نویسی بسیار پرکاربرد است.

⚙️ لیست کامل گزینه‌ها (OPTIONS)

  • -o [file] – ذخیره خروجی در یک فایل (با نام مشخص).
  • -O – ذخیره خروجی با نام فایل اصلی (از URL).
  • -L – دنبال کردن ریدایرکت (Follow redirects).
  • -I – فقط هدرهای پاسخ را نمایش می‌دهد (HEAD request).
  • -H [header] – اضافه کردن یک هدر سفارشی به درخواست.
  • -d [data] – ارسال داده با متد POST.
  • -X [method] – تنظیم متد درخواست (GET, POST, PUT, DELETE, ...).
  • -u [user:pass] – احراز هویت با نام کاربری و رمز عبور.
  • -k – نادیده گرفتن خطاهای SSL (غیرامن).
  • -v – نمایش جزئیات کامل (verbose).
  • -s – حالت بی‌صدا (silent) – نمایش خروجی کمتر.
  • -A [user-agent] – تنظیم User-Agent.
  • --help – نمایش راهنمای کوتاه.
  • --version – نمایش نسخه‌ی دستور.

🧪 مثال‌ها (EXAMPLES)

مثال ۱: دریافت محتوای یک وب‌سایت

curl example.com
خروجی: 
<!doctype html>
<html>
<head>
    <title>Example Domain</title>
    ...
</body>
</html>

محتوای HTML وب‌سایت example.com را در ترمینال نمایش می‌دهد.

مثال ۲: دانلود یک فایل

curl -O https://example.com/file.zip
خروجی: 
  % Total    % Received % Xferd  Average Speed   Time    Time     Time  Current
                                 Dload  Upload   Total   Spent    Left  Speed
100   100  100   100    0     0   1000      0 --:--:-- --:--:-- --:--:--  1000

فایل file.zip را با همان نام در پوشه‌ی فعلی دانلود می‌کند.

مثال ۳: دریافت فقط هدرها

curl -I google.com
خروجی: 
HTTP/1.1 301 Moved Permanently
Location: https://www.google.com/
Content-Type: text/html; charset=UTF-8
...

فقط هدرهای پاسخ سرور را نمایش می‌دهد (بدون محتوا).

مثال ۴: ارسال داده با متد POST

curl -X POST -d "name=ali&age=25" https://example.com/api
خروجی: 
(پاسخ سرور به درخواست POST)

داده‌های name=ali&age=25 را با متد POST به آدرس /api ارسال می‌کند.

مثال ۵: ذخیره خروجی با نام دلخواه

curl -o myfile.html example.com
خروجی: 
  % Total    % Received % Xferd  Average Speed   Time    Time     Time  Current
                                 Dload  Upload   Total   Spent    Left  Speed
100  1256  100  1256    0     0   5000      0 --:--:-- --:--:-- --:--:--  5000

محتوای example.com را در فایل myfile.html ذخیره می‌کند.

🧠 نکته (BUGS & NOTES)

نکته: curl به‌طور پیش‌فرض محتوا را در ترمینال نمایش می‌دهد. برای ذخیره در فایل از -o یا -O استفاده کنید.

نکته: برای ریدایرکت‌ها از -L استفاده کنید.

نکته: curl یک ابزار بسیار قدرتمند است و گزینه‌های بسیار زیادی دارد. برای اطلاعات کامل، man curl را ببینید.

✍️ تمرین

تمرین ۱: با curl محتوای یک وب‌سایت را دریافت کنید.

تمرین ۲: با curl -I هدرهای یک وب‌سایت را ببینید.

تمرین ۳: با curl -o محتوای یک وب‌سایت را در یک فایل ذخیره کنید.

تمرین ۴: با curl -L یک URL که ریدایرکت دارد را دنبال کنید.
